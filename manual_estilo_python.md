# Manual de Estilo para Creaci√≥n de Funciones en Python (Actualizado)

Este manual establece una gu√≠a detallada para escribir funciones en Python de forma **clara, modular, trazable y consistente**. La idea es que, al seguir estas directrices, se garantice la facilidad de mantenimiento, la reutilizaci√≥n del c√≥digo y una depuraci√≥n eficaz.

---

## 1. Nomenclatura: De lo General a lo Espec√≠fico

### 1.1. Nombres de Funciones
- **Formato:** Utiliza *snake_case* (todo en min√∫sculas y separado por guiones bajos).
- **Estructura sugerida:**  
  **verbo** + **objeto/√°mbito** + **detalle (opcional)** + **resultado** + **tipo de dato**  
  *Ejemplos:*  
  - `SQL_generate_report_str()`  
  - `API_fetch_data_dic()`  
  - `leads_calculate_conversion_float()`
- **Siglas reconocidas:** Escribe las siglas (por ejemplo, SQL, API, JSON) en **MAY√öSCULAS**.

### 1.2. Nombres de Variables
- Sigue la misma estructura que en las funciones, incluyendo un **sufijo de tipo**:
  - `_str`, `_int`, `_float`, `_bool`, `_dic`, `_list`, `_df`, `_set`, `_tuple`.
- *Ejemplos:*  
  - `campaign_cost_total_float`  
  - `SQL_query_str`  
  - `user_data_dic`

---

## 2. Estructura de Funciones

### 2.1. Definici√≥n y Argumentos
- **Uso de un √∫nico argumento:**  
  Todas las funciones deber√°n recibir un √∫nico par√°metro, generalmente un diccionario `params: dict`.  
- Si se requieren varios par√°metros, agr√∫palos en un diccionario para mantener la uniformidad y facilitar la validaci√≥n.
- **Docstrings completos:**  
  Cada funci√≥n debe incluir un docstring que contenga:
  - Una breve descripci√≥n de la funci√≥n.
  - Detalle de cada par√°metro en `params` (nombre, tipo y descripci√≥n).
  - Tipo y descripci√≥n del valor de retorno.
  - Posibles excepciones (`Raises`).

### 2.2. Ejemplo Base
```python
# @title SQL_generate_report_str()
def SQL_generate_report_str(params: dict) -> str:
    """
    Genera un reporte SQL en forma de cadena.

    Args:
        params (dict):
            - query (str): Consulta SQL.
            - format (str, opcional): Formato de salida (default: 'csv').

    Returns:
        str: Reporte generado en formato cadena.

    Raises:
        ValueError: Si falta el par√°metro 'query'.
    """
    query_str = params.get('query')
    if not query_str:
        raise ValueError("[VALIDATION [ERROR ‚ùå]] Falta 'query' en params.")
    
    return f"Reporte para: {query_str}"
```

## 3. Validaci√≥n de Par√°metros

- **Inmediatamente al inicio de la funci√≥n**, valida que se hayan proporcionado todos los par√°metros obligatorios.  
- Utiliza mensajes de error claros y utiliza el prefijo `[VALIDATION [ERROR ‚ùå]]` para indicar problemas en la validaci√≥n.
- Ejemplo:
  ```python
  if leads_total_int == 0:
      raise ValueError("[VALIDATION [ERROR ‚ùå]] Leads no puede ser cero.")
  ```

---

## 4. Estilo de C√≥digo y Formato

### 4.1. Formato y Espaciado
- **Indentaci√≥n:** Utiliza 4 espacios por nivel.
- **Espaciado entre funciones:** Deja 2 l√≠neas en blanco entre funciones.
- **L√≠neas en blanco:** Usa una l√≠nea en blanco antes del `return` si hay l√≥gica previa para mejorar la legibilidad.

### 4.2. Importaciones
- **Importaciones globales:** Coloca las importaciones generales (librer√≠as est√°ndar o de terceros que se usan en varias funciones) al inicio del m√≥dulo.
- **Importaciones locales:** Importa dentro de la funci√≥n aquellas librer√≠as cuyo uso es espec√≠fico para esa funci√≥n. Esto ayuda a reducir dependencias globales y mejora la encapsulaci√≥n.

## 5. Seguimiento y Mensajes de Log

### 5.1. Mensajes Estructurados
Utiliza `print(..., flush=True)` para asegurar que los mensajes se impriman inmediatamente. Se deben usar mensajes con prefijos estandarizados para cada fase del proceso:

- **Inicio y finalizaci√≥n:**
  - `[START ‚ñ∂Ô∏è]` ‚Üí Inicio del proceso o tarea.
  - `[END [FINISHED üèÅ]]` ‚Üí Finalizaci√≥n exitosa.
  - `[END [FAILED ‚ùå]]` ‚Üí Finalizaci√≥n con errores.

- **Autenticaci√≥n:**
  - `[AUTHENTICATION [INFO‚ÑπÔ∏è]]` ‚Üí Informaci√≥n sobre el proceso de autenticaci√≥n.
  - `[AUTHENTICATION [SUCCESS ‚úÖ]]` ‚Üí Autenticaci√≥n completada con √©xito.
  - `[AUTHENTICATION [ERROR ‚ùå]]` ‚Üí Error durante la autenticaci√≥n.

- **Extracci√≥n:**
  - `[EXTRACTION [START ‚è≥]]` ‚Üí Inicio de la extracci√≥n de datos.
  - `[EXTRACTION [SUCCESS ‚úÖ]]` ‚Üí Extracci√≥n completada correctamente.
  - `[EXTRACTION [WARNING ‚ö†Ô∏è]]` ‚Üí Advertencias durante la extracci√≥n.
  - `[EXTRACTION [ERROR ‚ùå]]` ‚Üí Error durante la extracci√≥n.

- **Transformaci√≥n:**
  - `[TRANSFORMATION [START ‚ñ∂Ô∏è]]` ‚Üí Inicio de la transformaci√≥n de datos.
  - `[TRANSFORMATION [SUCCESS ‚úÖ]]` ‚Üí Transformaci√≥n realizada correctamente.
  - `[TRANSFORMATION [WARNING ‚ö†Ô∏è]]` ‚Üí Advertencia durante la transformaci√≥n.
  - `[TRANSFORMATION [ERROR ‚ùå]]` ‚Üí Error durante la transformaci√≥n.

- **Carga (Load):**
  - `[LOAD [START ‚ñ∂Ô∏è]]` ‚Üí Inicio de la carga de datos.
  - `[LOAD [SUCCESS ‚úÖ]]` ‚Üí Carga completada correctamente.
  - `[LOAD [WARNING ‚ö†Ô∏è]]` ‚Üí Advertencia durante la carga.
  - `[LOAD [ERROR ‚ùå]]` ‚Üí Error en la carga.

- **M√©tricas y Reporte:**
  - `[METRICS [INFO ‚ÑπÔ∏è]]` ‚Üí Informaci√≥n y estad√≠sticas del proceso.

### 5.2. Organizaci√≥n Visual
- **Separadores y Encabezados:**  
  Separa cada grupo de acciones o secciones con una l√≠nea en blanco y un encabezado visual (por ejemplo, usando "üîπüîπüîπ") para agrupar fases o bloques de c√≥digo dentro de la funci√≥n.

## 6. Manejo de Errores

- **Uso de bloques `try-except`:**  
  Envuelve operaciones sensibles (como descarga de archivos, importaci√≥n din√°mica, lectura de datos) en bloques `try-except` para capturar y registrar errores.
- **Mensajes de Error Claros:**  
  Acompa√±a los errores con mensajes que incluyan el prefijo correspondiente, p.ej., `[EXTRACTION [ERROR ‚ùå]]`, y proporciona detalles √∫tiles (por ejemplo, el c√≥digo de estado HTTP o parte del contenido de la respuesta).

---

## 7. Funciones Auxiliares (Subfunciones)

- **Definici√≥n interna:**  
  Siempre que la funcionalidad sea espec√≠fica de una funci√≥n "madre", define las funciones auxiliares (subfunciones) dentro de ella.  
- **Nomenclatura interna:**  
  Utiliza un gui√≥n bajo inicial (`_`) para indicar que son funciones internas y no deben formar parte de la API p√∫blica.
- **Ejemplo:**
  ```python
  def load_custom_libs(config_list: list) -> None:
      def _download_module_from_github(module_path: str) -> str:
          # L√≥gica para descargar el m√≥dulo...
          return temp_file_path
      # Resto de la funci√≥n...
  ```

---

## 8. Orden y Organizaci√≥n del C√≥digo

- **Orden Alfab√©tico:**  
  Dentro del repositorio, las funciones deben ordenarse alfab√©ticamente por su nombre. Esto facilita la b√∫squeda y el mantenimiento.
- **Agrupaci√≥n L√≥gica:**  
  Las funciones relacionadas o que comparten un √°mbito similar deben agruparse en m√≥dulos o secciones dentro del archivo.

---

## 9. Consideraciones Adicionales

- **Actualizaci√≥n y Recarga de M√≥dulos:**  
  En funciones que realizan importaciones din√°micas (por ejemplo, `load_custom_libs`), se debe eliminar la versi√≥n previa del m√≥dulo y usar `importlib.invalidate_caches()` para asegurar la carga de la versi√≥n actual.
- **Gesti√≥n de Archivos Temporales:**  
  Cuando se descargue un archivo o m√≥dulo de forma din√°mica (por ejemplo, desde GitHub), utiliza la librer√≠a `tempfile` para guardar el archivo y aseg√∫rate de gestionar su eliminaci√≥n o mantenimiento seg√∫n sea necesario.
- **Uso de Zonas Horarias:**  
  Para mostrar fechas (por ejemplo, la fecha de √∫ltima modificaci√≥n de un m√≥dulo), utiliza el m√≥dulo `zoneinfo` (o alternativas como `pytz`) para ajustar la zona horaria seg√∫n corresponda.

## 10. Ejemplo Completo de Funci√≥n con Buenas Pr√°cticas

```python
# @title load_custom_libs()
def load_custom_libs(config_list: list) -> None:
    """
    Carga din√°micamente uno o varios m√≥dulos a partir de una lista de diccionarios de configuraci√≥n.

    Cada diccionario debe incluir:
      - module_host (str): "GD" para rutas locales o "github" para archivos en GitHub.
      - module_path (str): Ruta local o URL al archivo .py.
      - selected_functions_list (list, opcional): Lista de nombres de funciones/clases a importar.
        Si est√° vac√≠o se importan todos los objetos definidos en el m√≥dulo.

    Para m√≥dulos alojados en GitHub, la URL se transforma a formato raw y se descarga en un archivo temporal.
    La fecha de √∫ltima modificaci√≥n se muestra ajustada a la zona horaria de Madrid.

    Retorna:
        None

    Raises:
        ValueError: Si faltan par√°metros obligatorios.
    """
    import os
    import sys
    import importlib
    import inspect
    import datetime
    from zoneinfo import ZoneInfo
    import tempfile
    import requests

    # ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ Subfunciones Auxiliares ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
    def _imprimir_encabezado(mensaje: str) -> None:
        print(f"\nüîπüîπüîπ {mensaje} üîπüîπüîπ\n", flush=True)

    def _download_module_from_github(module_path: str) -> str:
        if "github.com" in module_path:
            raw_url = module_path.replace("github.com", "raw.githubusercontent.com").replace("/blob", "")
        else:
            raw_url = module_path
        try:
            print(f"[EXTRACTION [START ‚ñ∂Ô∏è]] Descargando m√≥dulo desde GitHub: {raw_url}", flush=True)
            response = requests.get(raw_url)
            if response.status_code != 200:
                error_details = response.text[:200].strip()
                print(f"[EXTRACTION [ERROR ‚ùå]] No se pudo descargar el archivo desde {raw_url}. C√≥digo de estado: {response.status_code}. Detalles: {error_details}", flush=True)
                return ""
            temp_file = tempfile.NamedTemporaryFile(delete=False, suffix=".py")
            temp_file.write(response.content)
            temp_file.close()
            print(f"[EXTRACTION [SUCCESS ‚úÖ]] Archivo descargado y guardado en: {temp_file.name}", flush=True)
            return temp_file.name
        except Exception as e:
            print(f"[EXTRACTION [ERROR ‚ùå]] Error al descargar el archivo desde GitHub: {e}", flush=True)
            return ""

    def _get_defined_objects(module, selected_functions_list: list) -> dict:
        all_objects = inspect.getmembers(module, lambda obj: inspect.isfunction(obj) or inspect.isclass(obj))
        defined_objects = {name: obj for name, obj in all_objects if getattr(obj, "__module__", "") == module.__name__}
        if selected_functions_list:
            return {name: obj for name, obj in defined_objects.items() if name in selected_functions_list}
        return defined_objects

    def _get_module_mod_date(module_path: str) -> datetime.datetime:
        mod_timestamp = os.path.getmtime(module_path)
        mod_date = datetime.datetime.fromtimestamp(mod_timestamp, tz=ZoneInfo("Europe/Madrid"))
        return mod_date

    def _import_module(module_path: str):
        module_dir, module_file = os.path.split(module_path)
        module_name, _ = os.path.splitext(module_file)
        if module_dir not in sys.path:
            sys.path.insert(0, module_dir)
            print(f"[TRANSFORMATION [INFO ‚ÑπÔ∏è]] Directorio agregado al sys.path: {module_dir}", flush=True)
        if module_name in sys.modules:
            del sys.modules[module_name]
            print(f"[TRANSFORMATION [INFO ‚ÑπÔ∏è]] Eliminada versi√≥n previa del m√≥dulo: {module_name}", flush=True)
        try:
            print(f"[LOAD [START ‚ñ∂Ô∏è]] Importando m√≥dulo: {module_name}", flush=True)
            module = importlib.import_module(module_name)
            module = importlib.reload(module)
            print(f"[LOAD [SUCCESS ‚úÖ]] M√≥dulo '{module_name}' importado correctamente.", flush=True)
            return module, module_name
        except Exception as e:
            print(f"[LOAD [ERROR ‚ùå]] Error al importar el m√≥dulo '{module_name}': {e}", flush=True)
            return None, module_name

    def _print_module_report(module_name: str, module_path: str, mod_date: datetime.datetime, selected_objects: dict) -> None:
        print("\n[METRICS [INFO ‚ÑπÔ∏è]] Informe de carga del m√≥dulo:", flush=True)
        print(f"  - M√≥dulo: {module_name}", flush=True)
        print(f"  - Ruta: {module_path}", flush=True)
        print(f"  - Fecha de √∫ltima modificaci√≥n: {mod_date}", flush=True)
        if not selected_objects:
            print("  - [WARNING ‚ö†Ô∏è] No se encontraron objetos para importar.", flush=True)
        else:
            print("  - Objetos importados:", flush=True)
            for name, obj in selected_objects.items():
                obj_type = type(obj).__name__
                doc = inspect.getdoc(obj) or "Sin documentaci√≥n"
                first_line = doc.split("\n")[0]
                print(f"      ‚Ä¢ {name} ({obj_type}): {first_line}", flush=True)
        print(f"\n[END [FINISHED ‚úÖ]] M√≥dulo '{module_name}' actualizado e importado en globals().\n", flush=True)

    # ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ Proceso Principal ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
    _imprimir_encabezado("[START ‚ñ∂Ô∏è] Iniciando carga de m√≥dulos personalizados")
    for config in config_list:
        module_host = config.get("module_host")
        module_path = config.get("module_path")
        selected_functions_list = config.get("selected_functions_list", [])

        if module_host == "github":
            temp_module_path = _download_module_from_github(module_path)
            if not temp_module_path:
                continue
            module_path = temp_module_path

        if not os.path.exists(module_path):
            print(f"[VALIDATION [ERROR ‚ùå]] La ruta del m√≥dulo no existe: {module_path}", flush=True)
            continue

        importlib.invalidate_caches()
        mod_date = _get_module_mod_date(module_path)
        module, module_name = _import_module(module_path)
        if module is None:
            continue

        selected_objects = _get_defined_objects(module, selected_functions_list)
        globals().update(selected_objects)
        _print_module_report(module_name, module_path, mod_date, selected_objects)
```
